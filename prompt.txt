Refactor my Vite React project's frontend to match the aesthetic of the Supabase Dashboard, built mobile-first. Do not change any business logic, routing, or data fetching — only restyle the UI.

Design reference: https://supabase.com/dashboard

## Core Principle: Mobile-First

Every component and layout decision starts at 320px and scales up. Write base styles for mobile, then layer on md: (768px) and lg: (1024px) overrides. Never write desktop styles and patch them down — always build up.

## Target Aesthetic

Supabase Dashboard characteristics to replicate:
- **Color palette**: Near-black backgrounds (#0f0f0f / #1c1c1c), dark surface cards (#1a1a1a / #222), muted borders (#333 / rgba(255,255,255,0.08)), green accent (#3ecf8e), and subtle off-white text (#ededed / #a3a3a3 for secondary)
- **Typography**: Geist or Inter — clean, compact, sans-serif. Font sizes are small and tight (13–14px body, 12px labels, 11px captions)
- **Spacing**: Dense but breathable. Consistent 4px grid. Sections have 16–24px padding. Minimal vertical rhythm
- **Layout**:
  - **Mobile (< 768px)**: No sidebar. Full-width content. Bottom nav bar OR hamburger menu that opens a full-screen drawer sliding in from the left. Top header is 48px, shows logo + hamburger icon + avatar only
  - **Tablet (768px–1023px)**: Collapsed icon-only sidebar (56px wide), top header shows page title
  - **Desktop (≥ 1024px)**: Full fixed sidebar (220–240px), top header (~52px), main content max-width ~1100px centered with padding
- **Components**:
  - Cards: full-width on mobile with 0 horizontal margin, 8px border-radius, 1px dark border, no shadows
  - Tables: horizontally scrollable on mobile inside a scroll container — never let them break layout. Consider card-based list view as an alternative on small screens
  - Buttons: full-width on mobile by default, auto-width from md breakpoint. Dark outlined ghost as default, solid green for primary CTA only
  - Inputs: full-width always, dark background, 1px border, green focus ring glow
  - Sidebar / Drawer: on mobile, renders as a left drawer with overlay backdrop (closes on backdrop tap). Smooth 200ms slide-in transform transition
  - Sidebar nav items: 44px minimum tap target height on mobile, 36px on desktop. Active state has muted green tint + left accent bar
  - Badges: muted pill badges with colored status dots
  - Modals / sheets: full-screen on mobile, centered modal from md breakpoint
- **Icons**: lucide-react — stroke-width 1.5, 16px inline / 20px nav / 24px for mobile nav bar icons
- **Touch**: All interactive elements must meet 44×44px minimum touch target. Use padding to hit this without inflating visual size
- **Motion**: Minimal — 150ms opacity/translate on hover (desktop only via @media (hover: hover)). Drawer/modal use 200ms slide/fade. No animations that block interaction

## Implementation Instructions

1. Create a global CSS file (or update the existing one) with CSS custom properties for the full token set (colors, spacing, radii, font sizes, shadows)
2. Build or update a Layout component with three responsive states: mobile drawer, icon sidebar, full sidebar
3. Implement the mobile nav drawer with: overlay backdrop, slide-in animation, close on backdrop click or swipe, trap focus while open
4. Audit every component and page — restyle each using the token system, starting from the smallest viewport
5. Wrap all tables in an overflow-x: auto scroll container; optionally render a stacked card list below md
6. Ensure all modals/dialogs are full-screen sheets on mobile with a drag handle, standard modals on desktop
7. Replace any light-mode or colorful styles with the dark, muted palette

## Constraints
- Keep all existing component props, state, and logic untouched
- Use only Tailwind utility classes OR pure CSS — do not mix both unless Tailwind is already configured
- If Tailwind is used, extend tailwind.config.js with the color tokens and set the default breakpoint strategy to mobile-first (it already is by default — do not use max-width variants as the primary approach)
- Do not install any new UI component libraries — lucide-react is the only additional package allowed
- Test mental model: always ask "does this work at 320px?" before moving to larger breakpoints

Start by listing all the files you will touch, then proceed file by file.
